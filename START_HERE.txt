🔴 BLOCKING ISSUES (in priority order):

  #1. Selector disabled by default (⚡ QUICK FIX)
      File: Model/flow_matching_model.py, line 6210
      Change: use_selector=False → use_selector=True
      Impact: +10-15 km ADE (no retraining needed)
      Time: 10 minutes

  #2. Environment loss missing (🔴 CRITICAL)
      File: Model/flow_matching_model.py, lines 5389-6205
      Problem: Steering features in model, but NO loss term constraining usage
      Solution: Add l_steering = MSE(v_pred, env_driven_component)
      Impact: +20-30 km ADE
      Time: 3-5 hours + full retraining

  #3. CFM loss vs ADE metric mismatch (🔴 CRITICAL)
      File: Model/flow_matching_model.py, lines 5945-6205
      Problem: Loss optimizes per-step denoising, metric measures trajectory quality
      Solution: Add multi-step ADE loss on 10% of samples
      Impact: +20-40 km ADE (LARGEST SINGLE FIX)
      Time: 6-8 hours + expensive retraining (+20-30% time)

  #4. Selector t-distribution mismatch (🔧 MEDIUM)
      File: Model/flow_matching_model.py, lines 6107-6139
      Problem: Selector trained on t=0.5 (noisy), evaluated on t≈0 (clean)
      Solution: Retrain selector on full DDIM trajectory
      Impact: +5-8 km ADE
      Time: 4-5 hours + retraining

  #5. Hard threshold train/test mismatch (🔧 SMALL)
      File: Model/flow_matching_model.py, line 6306
      Problem: Training uses p70+p50, inference uses p70 only
      Solution: Persist per-sample loss to checkpoint, load at inference
      Impact: +3-5 km ADE
      Time: Medium complexity
